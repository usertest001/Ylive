# 小米音响 / 小爱同学语音换台说明

如何通过 **Home Assistant（家庭助手）** 配合 **小米音响、小爱同学**，语音控制局域网内 **Y影音** 换台。

更完整的 App 功能介绍见 [README.md](./README.md)。

---

## 1. 能做什么

对着小米音响说：

- 「小爱同学，播放中央一台」
- 「小爱同学，下一个台」
- 「小爱同学，上一个台」

电视上的 Y影音 会自动换台并进入直播播放（需提前配好直播源）。

**工作方式：**

```text
您说话 → 小爱同学 → Home Assistant → Y影音（电视）→ 换台
```

小爱不会直接控制 Y影音，需要 Home Assistant 把语音转成对电视的网络请求。

---

## 2. 使用前准备

| 项目 | 要求 |
|------|------|
| 设备 | 已安装 Y影音 的电视 / 盒子，以及小米音响 |
| 网络 | 电视、音响、Home Assistant **连同一 Wi‑Fi / 局域网** |
| Y影音 | 已打开，已添加直播源，频道列表能正常显示 |
| Home Assistant | 已接入小米音响，能创建「自动化」 |

**建议：** 先在电视上手动打开 Y影音，进一次直播，确认能正常选台，再配置语音。

---

## 3. 查看电视 IP 地址

1. 电视打开 Y影音 → 首页 → **推送**  
2. 用手机（同 Wi‑Fi）扫页面二维码，或记下显示的地址，形如：  
   `http://192.168.1.100:9876/`  
3. 其中 `192.168.1.100` 即为电视 IP，下文示例请换成您的实际 IP  

也可在路由器管理页查看电视 / 盒子的局域网 IP。

---

## 4. 先用手动方式试通（推荐）

配置小爱之前，建议先用手机浏览器验证：

1. 手机与电视同一 Wi‑Fi  
2. 浏览器打开 `http://<电视IP>:9876/`  
3. 展开 **「遥控换台」**  
4. 点「下一台」「上一台」或选一个频道播放  

若此处能正常换台，再写入 Home Assistant；若此处不通，请先排查网络与 App 是否已打开。

## 5. 接口说明（供 Home Assistant 调用）

Y影音 在电视本机提供局域网服务，默认端口 **9876**。  
以下地址均将 `192.168.1.100` 替换为您的电视 IP。

### 5.1 播放指定频道（语音换台最常用）

```text
http://192.168.1.100:9876/api/channel/play?name=中央一台
```

| 参数 | 说明 |
|------|------|
| `name` | 频道名称或您常说的叫法，如 `CCTV1`、`湖南卫视` |
| `group` | 可选，分组名；多个分组有同名频道时建议加上 |
| `index` | 可选，频道序号（见下节「频道列表」），**最稳定** |

**示例：**

```text
# 按名字
http://192.168.1.100:9876/api/channel/play?name=CCTV1

# 按分组 + 名字
http://192.168.1.100:9876/api/channel/play?group=央视频道&name=CCTV1综合

# 按序号（推荐固定收藏台）
http://192.168.1.100:9876/api/channel/play?index=0
```

成功时电视会换台并进入直播；失败时可能提示「未找到频道」。

**按名字找台的规则（了解即可）：**

- 会优先在您**当前正在看的分组**里找，减少切错台  
- 支持常见别名，如「中央一台」「央视一套」等（可在 App 设置里配置频道别名）  
- 没说「高清」时，若有高清版线路，一般优先选高清  

### 5.2 下一个 / 上一个台

```text
http://192.168.1.100:9876/api/channel/next
http://192.168.1.100:9876/api/channel/prev
```

对应语音：「下一个台」「上一个台」。  
顺序与 App 内「换台反转」「列表循环」等设置一致。

### 5.3 查询频道列表

```text
http://192.168.1.100:9876/api/channels
```

浏览器或 Home Assistant 访问后，返回 JSON 列表，每项含 `index`、`name`、`group`。  
编写「播放第 N 个台」类自动化时可对照此列表。

### 5.4 仅进入 / 退出直播页（不换台）

```text
http://192.168.1.100:9876/api/live/enter    # 进入直播页
http://192.168.1.100:9876/api/live/exit     # 回到首页
http://192.168.1.100:9876/api/live/status   # 查询是否在直播页
```

### 5.5 成功与失败

成功一般返回：

```json
{ "code": 0 }
```

失败示例：

```json
{ "code": 1, "message": "未找到频道：中央一台" }
```

常见提示：

| 提示 | 处理办法 |
|------|----------|
| 频道列表未加载 | 在电视上打开 Y影音，等首页加载完 |
| 未找到频道 | 核对频道名；试推送页「遥控换台」；配置别名或改用序号 |
| 应用未就绪 | 确认 Y影音 已启动 |

## 6. Home Assistant 配置示例

以下 IP 请改为您的电视地址。不同 HA 版本、小爱接入方式可能略有差异，**思路均为：语音触发 → 访问上述网址**。

### 6.1 添加远程命令（rest_command）

在 `configuration.yaml` 中加入：

```yaml
rest_command:
  ylive_play:
    url: "http://192.168.1.100:9876/api/channel/play"
    method: GET
    timeout: 10
    params:
      name: "{{ name }}"

  ylive_next:
    url: "http://192.168.1.100:9876/api/channel/next"
    method: GET
    timeout: 10

  ylive_prev:
    url: "http://192.168.1.100:9876/api/channel/prev"
    method: GET
    timeout: 10
```

保存后重启 Home Assistant，或在「开发者工具 → YAML」中重载 `rest_command`。

### 6.2 绑定小爱语音（示例）

根据您使用的小米 / 小爱集成插件，创建自动化。逻辑示例：

**播放中央一台：**

```yaml
automation:
  - alias: "小爱-播放中央一台"
    trigger:
      # 此处按您的小爱集成实际事件填写
      - platform: event
        event_type: xiaomi_voice_command
        event_data:
          command: "播放中央一台"
    action:
      - service: rest_command.ylive_play
        data:
          name: "中央一台"
```

**下一个台：**

```yaml
  - alias: "小爱-下一个台"
    trigger:
      - platform: event
        event_type: xiaomi_voice_command
        event_data:
          command: "下一个台"
    action:
      - service: rest_command.ylive_next
```

也可使用 Node-RED、快捷指令等能发 HTTP 请求的工具，效果相同。

### 6.3 电脑命令行测试（可选）

与 Home Assistant 效果相同，便于排查：

```bash
curl -G "http://192.168.1.100:9876/api/channel/play" --data-urlencode "name=中央一台"
curl "http://192.168.1.100:9876/api/channel/next"
```

## 7. 提高语音识别命中率

| 做法 | 说明 |
|------|------|
| 配置频道别名 | Y影音 **设置** 中维护别名，让「央视一套」对应列表里的正式名称 |
| 固定说法 | 自动化里写死常看频道，如「播放中央一台」→ `name=中央一台` |
| 用序号 | 常看台查 `/api/channels` 记下 `index`，语音映射到固定序号最稳 |
| 带分组名 | 多个源有同名台时，口语或参数中加分组，如「联通的 CCTV1」 |

## 9. 常见问题

**Q：小爱说了没反应？**  
- 电视上的 Y影音 是否已打开  
- HA、音响、电视是否同网  
- 手机推送页「遥控换台」能否控制（先排除 App 与网络问题）  
- HA 自动化触发条件是否与小爱实际上报一致（查看 HA 日志）

**Q：提示找不到频道？**  
- 在推送页查看频道列表名称是否一致  
- 配置频道别名，或改用 `index` 序号  
- 同名台多加 `group` 参数

**Q：切错台了？**  
- 仅说频道名时，会优先在当前分组里找；可改说更全的名称或指定分组  
- 模糊匹配时可能优先高清线路

**Q：换台后黑屏？**  
- 请更新到较新公测版；已在直播页时会直接换源，不应整页重建  
- 仍有问题请到 App **关于 → 项目主页** 反馈，注明版本号

**Q：端口访问不了？**  
- 确认 Y影音 在运行  
- 路由器关闭 AP 隔离  
- 电视勿开 VPN；部分系统需允许局域网访问
