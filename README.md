# Y影音 — 软件介绍与说明

---

## 一、软件简介

**Y影音**是一款面向 **Android 电视、电视盒子、车机（车载安卓）、手机、平板** 的直播播放应用。软件本身**不提供、不存储、不运营**任何音视频节目，仅作为播放工具，播放您自行添加或配置的第三方链接内容。

### 适用场景

- 在电视、车机上用遥控器或触屏选台、看节目单、切换线路
- 在手机上竖屏浏览频道、收藏常看节目
- 同一局域网内，用手机或电脑遥控电视 / 车机换台
- 通过 **小米音响 / 小爱同学** 语音换台（配合 Home Assistant 调用局域网 HTTP 接口）
- 将手机上的视频投屏到电视，或将电视画面投到其他 DLNA 设备
- 通过链接或本地文件播放指定地址
- 部分须经 **SOCKS5 / HTTP 代理** 才能访问的 HLS 线路（见 §3.5）

### 系统要求

| 项目     | 说明                                                       |
| -------- | ---------------------------------------------------------- |
| 系统版本 | Android 6.0 及以上                                         |
| 设备类型 | Android TV、电视盒子、**车机（车载安卓系统）**、手机、平板 |
| 网络     | 播放网络直播需稳定网络；局域网遥控、投屏需设备在同一网络   |

> **车机说明：** 适用于运行 Android 的车载中控屏、后装车机等设备，界面以**横屏**为主，支持触屏与方向键操作。部分车机需在系统设置中允许「未知来源安装」、自启动与后台运行；行车时请勿操作屏幕，请遵守当地交通法规。

当前版本为**公测版**，功能持续完善中。如遇问题可通过 **QQ 交流群 480988885** 或应用内「关于 → 项目主页」反馈。

## 二、首次使用

### 1. 安装

从官方或作者提供的渠道获取 APK 安装包，按系统提示完成安装。若系统提示「未知来源」，请在系统设置中允许该来源后重试。

### 2. 阅读使用须知

首次启动会显示**使用须知**，请仔细阅读。点击「已阅读并同意」后方可继续使用；若不同意，请退出并卸载。

### 3. 屏幕方向（手机 / 车机）

部分手机首次启动可选择**横屏**或**竖屏**，建议手机用户选择竖屏，之后可在「设置 → 应用」中修改。

**车机用户**一般保持**横屏**即可，与电视 / 盒子布局一致；若车机系统强制竖屏，可在「设置 → 应用」中按需调整。

### 4. 添加直播源

软件不会自动替您订阅任何频道列表。您需要自行准备合法的 M3U、TXT 等格式的直播源，并在 **设置 → IPTV 源** 中添加：

- 远程地址（URL）
- 本地文件
- 直接粘贴文本内容

添加并启用源后，返回首页进入「直播」即可加载频道。

### 5. 节目单（EPG，可选）

若需查看「正在播出 / 即将播出」，可在 **设置 → EPG 源** 中配置节目单地址。EPG 数据同样来自您配置的第三方链接，与直播源分别管理。

## 三、主要功能说明

### 3.1 首页

首页按用途分为模块入口，横屏下常见分组为：

| 分组     | 常见功能                                                 |
| -------- | -------------------------------------------------------- |
| **播放** | 直播、全部频道、收藏、播放历史、搜索、链接播放、本地播放 |
| **工具** | 多屏同播、推送、电视遥控器、设置、关于                   |

首次使用建议：**设置里配好直播源 → 进入直播**（界面示意见上文 [界面预览](#界面预览)）。

### 3.2 直播与选台

- 支持按**分组**浏览频道，遥控器上下左右切换焦点，确认键进入播放
- 播放页可打开**快捷操作栏**（换源、节目单、投屏、频道信息等）
- 播放中可展开**频道列表 + 节目单（EPG）**，按日期切换、查看当前播出与即将播出，部分节目支持预约提醒
- 支持**收藏**常看频道、查看**播放历史**
- 部分线路支持多备用地址，可在频道信息中切换

### 3.3 播放器

软件内置多种播放内核，可在设置中按场景选择或切换，例如：

| 内核       | 适用说明（简要）             |
| ---------- | ---------------------------- |
| ExoPlayer  | 系统推荐，适合常见 HLS 等    |
| IJK        | 兼容面较广，部分直播流更稳定 |
| VLC        | 协议覆盖广                   |
| MPV        | 高级场景、部分特殊编码       |
| 系统播放器 | 轻量，部分协议不支持         |

具体默认值与高级参数请在「设置 → 播放器」中查看。若某频道无法播放，可尝试切换内核或更换线路。

### 3.4 链接播放与本地播放

- **链接播放**：输入 `http(s)`、`rtsp` 等可播放地址直接观看
- **本地播放**：通过系统文件选择器播放设备上的音视频文件
- 其他应用可通过系统「打开方式」将链接交给 Y影音 播放（视系统与链接类型而定）

### 3.5 SOCKS5 / HTTP 代理播放

部分直播源（尤其部分爬虫类 TXT / M3U）的线路必须先经 **SOCKS5、SOCKS4 或 HTTP/HTTPS 代理** 才能访问真实播放地址。普通播放器无法直接为任意地址配置 SOCKS，Y影音 使用专用地址格式 **`ylive://play`**（并兼容简写 `sk://`、`socks://`、`socks5://`），由应用在本机代为走代理拉流，再交给各播放内核播放。

#### 适用场景

- 源里一行类似：`socks5://代理IP:端口&http://真实m3u8地址`
- 直连 m3u8 超时或无法播放，但经代理可访问

#### 地址写法

```text
# 标准格式（推荐）
ylive://play?proxy=<代理地址>&url=<URL编码后的真实播放地址>

# 简写格式（等价）
sk://<代理IP:端口>&url=<真实播放地址>
socks://<代理IP:端口>&url=<真实播放地址>
socks5://<代理IP:端口>&url=<真实播放地址>
```

| 参数    | 说明                                                         |
| ------- | ------------------------------------------------------------ |
| `proxy` | 代理地址，支持 `socks5://`、`socks4://`、`http://`、`https://` |
| `url`   | 真实 `http://` 或 `https://` 播放地址，建议 URL 编码（目标地址含 `&` 参数时应编码） |

**IPTV 源示例（可在频道行 URL 中使用）：**

```text
CCTV-6,ylive://play?proxy=socks5://1.2.3.4:1080&url=http%3A%2F%2Fexample.com%2Flive%2Fcctv6.m3u8$代理线路
```

`$` 后面为线路备注名，不影响解析。

**链接播放 / 手机推送页：** 可直接粘贴 `ylive://play?...` 或 `sk://...&url=...` 等简写。

#### 使用步骤

1. 确认代理服务器可用（IP、端口正确，电视/盒子能访问该代理）  
2. 将真实 m3u8 地址做 **URL 编码**（`:` → `%3A`，`/` → `%2F` 等）  
3. 按上表拼成 `ylive://play?proxy=...&url=...`（或简写 `sk://...&url=...`）  
4. 写入 IPTV 源对应频道，或通过 **链接播放**、**推送页** 测试  
5. 若仍无法播放，尝试切换 **IJK / VLC / ExoPlayer** 等内核  

#### 注意事项

| 项目          | 说明                                                         |
| ------------- | ------------------------------------------------------------ |
| 支持的目标    | 仅 **http / https**（常见 HLS m3u8）；**不支持**经代理的 RTSP、UDP |
| SOCKS 认证    | **无用户名密码**的 SOCKS5 最稳定；带账号密码的 SOCKS 可能无法使用 |
| HTTP 代理认证 | 可在代理地址中写 `http://用户名:密码@IP:端口`                |
| 播放表现      | 首帧可能较慢（需经代理拉取清单与分片），属正常现象           |
| 直连 CDN      | 部分 CDN 直连 HLS 不走代理中继；只有标注须代理的线路才需用 `ylive://play` |

#### 简单排错

1. 代理是否可达、端口是否开放  
2. `url=` 是否已正确 URL 编码  
3. 真实地址在电脑浏览器（经同一代理）能否打开  
4. 换播放内核再试  

### 3.6 DLNA 投屏

在 **设置 → 应用** 中可分别开启：

- **接收投屏**：本机作为电视被手机等设备投屏
- **投到其他设备**：将当前播放内容投到局域网内其他 DLNA 设备

使用前请确认电视与手机处于同一 Wi‑Fi，且路由器未禁止设备互访。

### 3.7 局域网遥控

#### 方式一：浏览器遥控（推荐在电脑使用）

电视 / 车机与手机/电脑在同一局域网时，在浏览器访问：

```text
http://<设备IP地址>:9876/advance
```

可进行换台、修改部分设置、链接播放等操作。设备 IP 可在路由器管理页或车机 / 电视网络设置中查看。

#### 方式二：App 内遥控

在首页进入 **电视遥控器**，输入目标设备的 IP 或扫描推送页上的二维码，连接后可远程换台、进入/退出直播页等。被控端须为同一网络下的 Y影音 设备。

#### 方式三：小米音响 / 小爱同学语音换台（Home Assistant）

Y影音 在局域网内提供 **HTTP 换台接口**（默认端口 **9876**），可与 **Home Assistant** 联动，再由 **小米音响、小爱同学** 语音控制电视 / 车机上的 Y影音 换台，例如：

- 「小爱同学，播放中央一台」
- 「小爱同学，下一个台 / 上一个台」

**前提条件：**

1. 电视 / 盒子 / 车机上的 Y影音 与小米音响、Home Assistant 处于**同一局域网**
2. 被控端 Y影音 **已启动**，且直播源、频道列表已加载（可先手动打开一次 App）  
3. 已在 Home Assistant 中接入小米音响，并会配置「调用 REST / 命令行」类自动化  

**常用接口（将 `192.168.x.x` 换成电视 IP）：**

| 操作              | 接口示例                                                     |
| ----------------- | ------------------------------------------------------------ |
| 按频道名播放      | `GET http://192.168.x.x:9876/api/channel/play?name=中央一台` |
| 按序号播放        | `GET http://192.168.x.x:9876/api/channel/play?index=0`       |
| 下一台            | `GET http://192.168.x.x:9876/api/channel/next`               |
| 上一台            | `GET http://192.168.x.x:9876/api/channel/prev`               |
| 进入 / 退出直播页 | `GET .../api/live/enter` · `GET .../api/live/exit`           |
| 查询频道列表      | `GET http://192.168.x.x:9876/api/channels`                   |

仅传频道名时，会**优先在当前播放所在分组**内匹配，减少误切到其他分组的同名频道；也支持在 **设置** 中配置频道别名，便于语音识别「央视一套」「CCTV1」等不同叫法。

**Home Assistant 配置思路（简述）：**

1. 在 HA 中为小米音响创建语音指令或自动化场景  
2. 触发时调用上述 HTTP 接口（可用 `rest_command`、Node-RED、shell_command + `curl` 等）  
3. 换台成功后，电视端会自动进入直播页并开始播放  

手机访问电视 **推送页**（`http://<电视IP>:9876/`）中的「遥控换台」面板，与上述接口相同，可先手动试通再写入 HA。

> 详细配置步骤与接口说明见 [小米音响小爱同学语音换台接口文档.md](./小米音响小爱同学语音换台接口文档.md)。

### 3.8 推送页

在首页进入 **推送**，可用手机扫描页面二维码，向电视发送链接或进行简单控制（具体以页面提示为准）。

### 3.9 多屏同播

在同一台设备上分屏同时播放多个频道，适合监控或多路对比场景（性能受设备限制）。

### 3.10 云同步（可选）

可将频道分组、收藏、部分设置等备份到：

- 网络链接
- WebDAV
- 本地文件
- 其他支持的云存储方式

在 **设置 → 云同步** 中配置。同步前请妥善保管账号与链接，避免泄露个人隐私。

### 3.11 启动画面

可在 **设置 → 应用** 中配置开机启动页、横竖屏启动图、是否使用必应每日壁纸等。

### 3.12 检查更新

在 **关于** 页点击「检查更新」可查询是否有新版本。连续快速双击可触发强制检查（以实际版本策略为准）。

## 五、使用建议与常见问题

### 5.1 无法播放或频繁卡顿

1. 检查网络是否正常，尝试切换 Wi‑Fi / 有线网络  
2. 在频道信息中**更换线路**  
3. 在设置中**更换播放器内核**  
4. 确认直播源地址是否仍有效（源失效时软件无法凭空恢复节目）

### 5.2 代理线路（ylive://play）播不了

1. 检查 `proxy` 地址是否正确，代理服务是否在线  
2. 确认 `url=` 后的真实地址已 **URL 编码**  
3. 代理线路仅支持 **http/https** 的 HLS，不支持 RTSP/UDP 走代理  
4. 尽量使用 **无认证** 的 SOCKS5；HTTP 代理可用 `user:pass@` 形式  
5. 在 **链接播放** 中单独测试整条 `ylive://play` 或 `sk://` 地址，再写入 IPTV 源  
6. 尝试更换播放内核（IJK、VLC、ExoPlayer）

### 5.3 没有节目单

EPG 需单独配置且与频道名称匹配；未配置或匹配不上时，界面可能不显示节目信息，但不影响播放。

### 5.4 局域网遥控连不上

1. 确认两台设备在同一局域网  
2. 确认电视未关闭 HTTP 服务（一般随应用运行）  
3. 检查路由器是否开启「AP 隔离」  
4. 防火墙或定制系统可能拦截 9876 端口

### 5.5 小爱 / 小米音响语音换台无反应

1. 确认电视端 Y影音 **已打开**，频道列表已加载（可先手动进一次直播）  
2. 确认 Home Assistant、小米音响、电视 **在同一局域网**，且 HA 能访问 `http://<电视IP>:9876/api/channels`  
3. 频道名尽量与列表一致，或在设置中配置**频道别名**（如「中央一台」→「CCTV1综合」）  
4. 仅说频道名时，会优先在**当前分组**内查找；可改用 `index` 序号换台更稳定  
5. 先在手机浏览器打开推送页「遥控换台」试播，成功后再写入 HA 自动化

### 5.6 投屏搜不到设备

确认 DLNA 相关开关已打开，且手机与电视同网；部分设备需关闭 VPN 后再试。

### 5.7 开机自启无效

不同品牌电视 / 盒子 / 车机对后台与自启动限制不同，除应用内开启外，还须在**系统设置**中允许本应用自启动、后台运行，并关闭过度省电限制。车机若安装后无法自启或被杀后台，请在车机「应用管理」中为 Y影音 开启自启动、后台活动与省电白名单。

### 5.8 崩溃或异常

若出现闪退，可留意是否弹出崩溃提示页，按提示**报送异常**（需网络），便于后续版本修复。软件可能上报设备型号、版本号及崩溃日志，**不含播放内容与频道列表**。

## 六、隐私与数据说明
